# Productivity Dashboard

Offline analytics dashboard for the AI CI/CD system's productivity analysis data.

## Directory Contents

| File | Purpose |
|---|---|
| `export_firestore_data.py` | Exports productivity events + workflow metadata from Firestore to JSON |
| `productivity_data.json` | Exported data (generated by the script above — gitignored) |

## Workflow

### 1. Export data from Firestore

```bash
cd productivity
pip install google-cloud-firestore   # if not already installed
gcloud auth application-default login
python export_firestore_data.py
```

This produces `productivity_data.json` containing:
- All `productivity_analysis` events (time breakdowns, cost savings)
- Workflow metadata (status, repo, branch, timestamps)
- Event type distribution summary

### 2. Build the dashboard

Upload `productivity_data.json` to Claude and use the interactive React dashboard artifact.
The dashboard is designed for screen-sharing during the exhibition presentation.

### 3. Key metrics displayed

- **Time Saved**: Human estimate vs AI resolution time per workflow
- **Cost Savings**: Dollar value at $75/hr loaded rate
- **Effort Breakdown**: Codebase comprehension, CI triage, diagnosis, fix implementation, etc.
- **Trends**: Performance across multiple workflow runs

## Data Integrity

The productivity analysis uses an **anti-bias methodology**:
- Claude classifies observable actions (what happened), never estimates its own value
- Time values come from `PRODUCTIVITY_TIME_MAP` — hardcoded constants backed by published research
- See `prompts/productivity_rubric_research.md` for full citations
